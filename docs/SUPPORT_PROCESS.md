# Регламент обработки обращений (поддержка)

Документ для операторов поддержки и админов, обрабатывающих тикеты из раздела «Обратная связь».

## Статусы тикета

- **open** — новое обращение, ожидает первичного ответа.
- **in_progress** — обращение взято в работу (ответили или начали разбирательство).
- **resolved** — вопрос решён, ответ передан пользователю.

## Сроки

- Первичный ответ пользователю — в течение **24 часов** (рабочих) с момента поступления обращения.
- При необходимости уточнения информации — запросить у пользователя (телефон, email из тикета или контакт из сообщения).

## Как связаться с пользователем

- В тикете указаны: **contact_phone** (обязателен для анонимных обращений), при авторизованном пользователе — данные из профиля.
- Ответ можно передать по телефону, в мессенджере или по email (если пользователь указал в сообщении). При смене статуса на «resolved» при необходимости заполнить **admin_notes** (например: «Связались по телефону, возврат оформлен»).

## Шаблоны ответов (примеры)

- **Вопрос по заказу:** «Здравствуйте! Ваш заказ №… принят в работу. Ожидаемая дата доставки… По вопросам отслеживания обращайтесь по этому номеру.»
- **Доставка:** «Информация по доставке заказа №…: … При изменении сроков мы свяжемся с вами.»
- **Возврат/претензия:** «Для оформления возврата необходимо … Срок рассмотрения заявки до 5 рабочих дней. Мы свяжемся с вами по указанному контакту.»

## Эскалация

- При повторных жалобах, юридических претензиях или сбоях в работе сервиса — передать ответственному за продукт/инфраструктуру (см. контакты в DISASTER_RECOVERY.md или внутренний регламент).
