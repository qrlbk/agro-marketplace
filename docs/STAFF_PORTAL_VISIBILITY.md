# Кто что видит в портале сотрудников

Матрица видимости разделов по ролям: Support, Admin, Super Admin.

## Таблица «Роль × Раздел»

| Раздел            | Support | Admin | Super Admin |
|-------------------|--------|-------|-------------|
| Дашборд           | —      | ✓     | ✓           |
| Заказы            | —      | ✓     | ✓           |
| Поставщики        | —      | ✓     | ✓           |
| Пользователи      | ✓ (только просмотр) | ✓ | ✓           |
| Обращения         | ✓      | ✓     | ✓           |
| Журнал действий   | —      | ✓     | ✓           |
| Поиск             | ✓      | ✓     | ✓           |
| Сотрудники        | —      | —     | ✓           |
| Роли              | —      | —     | ✓           |
| Профиль           | ✓      | ✓     | ✓           |

## Поведение по ролям

- **Support:** только «Обращения», «Пользователи» (просмотр), «Поиск», «Профиль». После входа — редирект на первый доступный раздел (список обращений). Пользователи нужны для поиска клиента по телефону при разборе обращения.
- **Admin:** Дашборд, Заказы, Поставщики, Пользователи, Обращения, Журнал действий, Поиск, Профиль. Без разделов «Сотрудники» и «Роли».
- **Super Admin:** все разделы, включая «Сотрудники» и «Роли».

## Соответствие прав и разделов

- Дашборд — `dashboard.view`
- Заказы — `orders.view`
- Поставщики — `vendors.view`
- Пользователи — `users.view`
- Обращения — `feedback.view`
- Журнал действий — `audit.view`
- Поиск — `search.view`
- Сотрудники — `staff.manage`
- Роли — `roles.manage`
- Профиль — без проверки (доступен всем вошедшим)

При отсутствии права на страницу пользователь перенаправляется на первый разрешённый раздел.
